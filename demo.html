Interactive HTTP Requests in Markdown

Like mermaid.js, but for HTTP requests. Compatible with VSCode REST Client format.

1. Simple GET Request

Basic GET request to a public API. Click "Send" to execute.

### GET https://api.github.com/users/octocat Accept: application/json

2. POST Request with JSON Body

POST request with JSON payload. Try it with JSONPlaceholder API.

### # @name createPost POST https://jsonplaceholder.typicode.com/posts Content-Type: application/json { "title": "Hello from HttpRex", "body": "This is a test post created with httprex", "userId": 1 }

3. Request with Variables

Using file variables and system variables. The variables are highlighted in orange.

@baseUrl = https://jsonplaceholder.typicode.com ### # @name getUser GET {{baseUrl}}/users/1 Accept: application/json X-Request-ID: {{$guid}} X-Timestamp: {{$timestamp}}
‚ö†Ô∏è Variable Warning: Unresolved variables will show a warning. Define them using @varName = value syntax.

4. Multiple HTTP Methods

Different HTTP methods with color-coded badges.

### # @name listUsers GET https://jsonplaceholder.typicode.com/users ### # @name updatePost PUT https://jsonplaceholder.typicode.com/posts/1 Content-Type: application/json { "id": 1, "title": "Updated Title", "body": "Updated content", "userId": 1 } ### # @name deletePost DELETE https://jsonplaceholder.typicode.com/posts/1

5. Custom Headers

Request with multiple custom headers.

### GET https://httpbin.org/headers User-Agent: HttpRex/1.0 Accept: application/json Accept-Language: en-US,en;q=0.9 Cache-Control: no-cache X-Custom-Header: test-value

6. Form URL Encoded

POST request with form-urlencoded data (fixed: no more double-encoding!).

### POST https://httpbin.org/post Content-Type: application/x-www-form-urlencoded name=John Doe email=john@example.com message=Hello from httprex!

7. Error Handling

Test error scenarios (404, network errors, etc.).

### GET https://api.github.com/users/this-user-definitely-does-not-exist-12345
üí° Tip: HttpRex shows detailed error responses including status codes, timing, and error messages.

üìö How to Use

Instructions for using HttpRex in your own projects.

<!-- Basic Usage --> <httprex-block> ### GET https://api.example.com/endpoint Authorization: Bearer your-token </httprex-block> <!-- With Variables --> <httprex-block> @apiUrl = https://api.example.com @token = your-secret-token ### GET {{apiUrl}}/users Authorization: Bearer {{token}} </httprex-block> <!-- Include the library --> <script type="module" src="path/to/httprex.js"></script>

Programmatic API

import HttpRex from 'httprex'; // Parse request const result = HttpRex.parse(` GET https://api.example.com/data `); // Execute request const response = await HttpRex.execute(result.data); console.log(response); // Auto-discover and init all blocks HttpRex.init({ selector: '.language-httprex' });

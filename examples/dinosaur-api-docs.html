Overview

RESTasaurus provides comprehensive data on almost 1,200 dinosaur species. Query dinosaurs by name, diet, clade, locomotion type, and more.

API Configuration

Set these variables once and use them throughout all requests:

@baseUrl = https://restasaurus-api.vercel.app @version = v1 ### Configuration Check # @name healthCheck GET {{baseUrl}}/api/{{version}}/
Rate Limiting The API is rate limited to 20 requests per hour. For higher limits, consider self-hosting.

Authentication

RESTasaurus is a public API that does not require authentication. All endpoints are freely accessible via GET requests.

No API Key Required Simply make requests to the endpoints below. No signup or tokens needed!

Dinosaurs

Core endpoints for retrieving dinosaur data. Filter, search, and explore the prehistoric world.

GET /api/v1/dinosaurs

Retrieve a paginated list of all dinosaurs. Supports filtering by various attributes.

Query Parameters

Parameter Type Description
limit integer Maximum number of results (default: 10, max: 100)
offset integer Number of results to skip for pagination
Try it
@baseUrl = https://restasaurus-api.vercel.app ### # @name listAllDinosaurs # Get the first 5 dinosaurs GET {{baseUrl}}/api/v1/dinosaurs?limit=5 Accept: application/json
GET /api/v1/dinosaurs/:id

Retrieve detailed information about a specific dinosaur by its unique ID.

Path Parameters

Parameter Type Description
id integer required The unique identifier of the dinosaur
Try it
@baseUrl = https://restasaurus-api.vercel.app @dinoId = 1 ### # @name getDinosaurById GET {{baseUrl}}/api/v1/dinosaurs/{{dinoId}} Accept: application/json
GET /api/v1/dinosaurs/name/:name

Look up a dinosaur by its name. Names are case-insensitive.

Try it
@baseUrl = https://restasaurus-api.vercel.app ### # @name getDinosaurByName # Look up the famous T-Rex GET {{baseUrl}}/api/v1/dinosaurs/name/tyrannosaurus Accept: application/json
GET /api/v1/dinosaurs/diet/:diet

Filter dinosaurs by their dietary classification.

Available Diets

  • herbivore - Plant eaters
  • carnivore - Meat eaters
  • omnivore - Both plants and meat
Try it
@baseUrl = https://restasaurus-api.vercel.app ### # @name getCarnivores # Get all meat-eating dinosaurs GET {{baseUrl}}/api/v1/dinosaurs/diet/carnivore Accept: application/json
GET /api/v1/dinosaurs/random

Get a random selection of dinosaurs. Perfect for "Dinosaur of the Day" features!

Query Parameters

Parameter Type Description
count integer Number of random dinosaurs to return (default: 1, max: 10)
Try it
@baseUrl = https://restasaurus-api.vercel.app ### # @name getRandomDinosaurs # Get 3 random dinosaurs for a quiz GET {{baseUrl}}/api/v1/dinosaurs/random?count=3 Accept: application/json X-Request-ID: {{$guid}}
GET /api/v1/dinosaurs/search

Advanced search with multiple filter criteria.

Query Parameters

Parameter Type Description
clade string Filter by taxonomic clade (e.g., Theropoda, Sauropoda)
diet string Filter by diet type
locomotion string Filter by locomotion type (bipedal, quadrupedal)
Try it
@baseUrl = https://restasaurus-api.vercel.app ### # @name searchDinosaurs # Find bipedal carnivores (like raptors) GET {{baseUrl}}/api/v1/dinosaurs/search?diet=carnivore&locomotion=bipedal Accept: application/json

Error Handling

The API uses standard HTTP status codes to indicate success or failure.

Status Code Meaning
200 OK Request successful
400 Bad Request Invalid request parameters
404 Not Found Dinosaur or resource not found
429 Too Many Requests Rate limit exceeded (20 req/hour)
500 Internal Server Error Server error - please try again

Error Response Format

{
  "error": "Dinosaur not found",
  "status": 404,
  "message": "No dinosaur with name 'unicornosaurus' exists"
}
Test a 404 Error
@baseUrl = https://restasaurus-api.vercel.app ### # @name test404Error # This will return a 404 error GET {{baseUrl}}/api/v1/dinosaurs/name/unicornosaurus Accept: application/json

Complete Workflow Example

Here's a complete workflow showing how to explore the API: first get metadata, then search for specific dinosaurs.

@baseUrl = https://restasaurus-api.vercel.app @diet = carnivore ### # Step 1: Check available clades # @name step1_getClades GET {{baseUrl}}/api/v1/clades Accept: application/json ### # Step 2: Search for theropod carnivores # @name step2_searchTheropods GET {{baseUrl}}/api/v1/dinosaurs/search?clade=theropoda&diet={{diet}} Accept: application/json ### # Step 3: Get details on a specific dinosaur # @name step3_getDetails GET {{baseUrl}}/api/v1/dinosaurs/name/velociraptor Accept: application/json ### # Step 4: Get a random dinosaur for comparison # @name step4_getRandom GET {{baseUrl}}/api/v1/dinosaurs/random Accept: application/json X-Request-ID: {{$guid}} X-Timestamp: {{$timestamp}}
